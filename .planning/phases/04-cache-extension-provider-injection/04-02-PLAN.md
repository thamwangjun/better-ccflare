    - When 4 cache_control blocks already exist across the request, NO additional injection occurs (total stays exactly 4)
    - When 2 cache_control blocks already exist, inject only 2 more (remaining = 4 - 2 = 2), not 3 or 4

    D-01 — Non-destructive guard:
    - A block that already has cache_control is left completely untouched (existing value preserved, not overwritten)

    - Stored JSON without allow_fallbacks key → injected allow_fallbacks: true
