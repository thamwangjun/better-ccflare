### E2E Flow: Set Provider Preference

1. Operator opens dropdown on an OpenRouter account → `account.provider === "openrouter"` gate in `AccountListItem.tsx` (line 251) shows "Provider Preferences" item
2. Dialog opens pre-populated via `syncProviderPreferenceState(account)` reading `account.openrouterProviderPreference.order`
3. Operator enters comma-separated providers → `parseProviderOrder()` converts to string array
4. Save → `resolveProviderPreferenceSaveAction()` branches to SET → `api.putAccountOpenrouterProviderPreference(id, order, allowFallbacks)` → PUT /api/accounts/:id/openrouter-provider-preference
5. Server stores JSON `{ order: [...], allow_fallbacks: true }` via `setAccountOpenrouterProviderPreference(id, jsonString)`
6. Next proxy request for this account: `transformRequestBody()` reads `account.openrouter_provider_preference`, parses it, injects `body.provider = { order, allow_fallbacks }` when no client `provider` field present
7. `loadAccounts()` called after save — UI reflects new state

**Status: COMPLETE**

### E2E Flow: Clear Provider Preference

1. Operator opens dialog with preference set → `parseProviderOrder("")` returns `[]`
2. Save with empty order → `resolveProviderPreferenceSaveAction()` branches to CLEAR → `api.deleteAccountOpenrouterProviderPreference(id)` → DELETE /api/accounts/:id/openrouter-provider-preference
3. Server sets `openrouter_provider_preference = NULL` via `setAccountOpenrouterProviderPreference(id, null)`
4. Next proxy request: `account.openrouter_provider_preference` is null → provider injection block skipped → no `body.provider` injected

**Status: COMPLETE**
