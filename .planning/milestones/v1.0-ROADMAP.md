## Overview

Established the fork as a stable, maintainable base by fixing two silent correctness bugs in the OpenRouter provider (billing underreporting and broken cache injection) and building operational tooling for safe upstream merges. All 7 v1 requirements delivered in 2 days across 35 commits.

## Phases

### Phase 1: Correctness & Patch Hardening

**Goal:** The proxy accurately reports OpenRouter cache token costs and all fork patches are identifiable and test-covered so they survive upstream merges without silent regression.
**Depends on:** Nothing (first phase)
**Requirements:** CACHE-01, CACHE-02, PATCH-01, PATCH-02
**Plans:** 3 plans

Plans:
- [x] 01-01: Fix OpenRouterProvider: add extractUsageInfo override (CACHE-01) and 3-breakpoint per-block cache injection (CACHE-02)
- [x] 01-02: Add FORK PATCH comment to cacheCreationInputTokens in openai/provider.ts (PATCH-01)
- [x] 01-03: Update OpenRouter tests: per-block assertions + extractUsageInfo regression guard (CACHE-02 / PATCH-02)

**Details:**

Two correctness bugs fixed and hardened with regression tests:

*CACHE-01 — extractUsageInfo override:* OpenRouterProvider previously inherited `BaseAnthropicCompatibleProvider.extractUsageInfo()`, which reads `usage.cache_creation_input_tokens` — a field Anthropic returns but OpenRouter does not. Override now reads `usage.prompt_tokens_details.cache_write_tokens` → `cacheCreationInputTokens` and `usage.prompt_tokens_details.cached_tokens` → `cacheReadInputTokens`. Streaming path delegates to parent (unchanged). Wrapped in try/catch — malformed upstream responses return null.

*CACHE-02 — 3-breakpoint per-block injection:* Replaced broken top-level `body.cache_control = { type: "ephemeral" }` (not a valid Anthropic/OpenRouter field) with per-block injection at: (1) last tool in `body.tools[]`, (2) last system content block (string system converted to array), (3) last assistant turn in `body.messages[]`.

*PATCH-01:* Added `// FORK PATCH: cache_write_tokens from prompt_tokens_details (OpenRouter)` comment above `cacheCreationInputTokens` declaration in `openai/provider.ts` — visible during upstream diffs.

*PATCH-02:* 10-test suite in `openrouter/__tests__/provider.test.ts` — 4 tests for extractUsageInfo (regression guard: `expect(usage?.cacheCreationInputTokens).toBe(50)`) and 6 tests for transformRequestBody per-block assertions.

TDD commits: RED `c3661ec`, GREEN `db5e5d7`.

**Depends on:** Phase 1
**Requirements:** MAINT-01, MAINT-02, MAINT-03
**Plans:** 1 plan

Plans:

**Details:**

Three deliverables providing an end-to-end upstream merge workflow:

*pre-merge-check.sh (MAINT-01):* Hardcodes 3 high-risk files (`openai/provider.ts`, `openrouter/provider.ts`, `types/src/account.ts`), guards for `upstream/main` remote, prints `git diff` + fork commit log per file using color-auto. Run via `bun run pre-merge-check`.

Run via `bun run post-merge-export`. Patch files gitignored (generated artifacts).

*UPSTREAM_MERGE.md:* 6-step agent-executable SOP (fetch → inspect → merge → resolve → export → push) with per-file conflict resolution notes for all 3 high-risk files including exact FORK PATCH search anchors, resolution rules, and 2 edge cases.

Auto-fixed during execution: gitignore exception for `.planning/scripts/*.sh` (blocked by `**/*.sh`), Bash 3.2 last-line fix for `git log --pretty=format:` (missing trailing newline), gitignore for generated patch exports.

## Milestone Summary

**Key Decisions:**

- Override `extractUsageInfo` in `OpenRouterProvider` rather than modify base class — OpenRouter's `prompt_tokens_details` format is non-standard; modifying the base would break Anthropic/Bedrock paths
- Delegate streaming path to `super.extractUsageInfo()` — streaming usage extraction is unchanged; only non-streaming JSON format differs
- Convert string system to `[{type,text,cache_control}]` array — Anthropic breakpoint 2 requires block-level injection; strings have no block structure
- Hardcode high-risk file list in `pre-merge-check.sh` (not dynamic scanning) — simpler and stable for a 3-file surface
- Patch files in `.planning/patches/` are gitignored — generated artifacts regenerated on every run; git tags provide durable commit references
- Fork upstream rather than contribute all changes back — some changes (infra, personal config) aren't appropriate for upstream; faster iteration

**Issues Resolved:**

- Operator billing display understated cache write tokens by 5x (OpenRouter `prompt_tokens_details` vs Anthropic `cache_creation_input_tokens`)
- Top-level `cache_control` injection silently failed for all Bedrock/Vertex routes
- No safe upstream merge process — patches could be silently dropped during merges

**Issues Deferred:**

- SC-2 (CACHE-02): Live non-Anthropic model request test deferred — human verification: send `z-ai/glm-4.5-air:free` via proxy to OpenRouter account and confirm 2xx with no 400 errors
- REQUIREMENTS.md CACHE-02 wording vs implementation mismatch (cosmetic only)
- Phase 2 has no VERIFICATION.md (VALIDATION.md + SUMMARY.md are present; nyquist_compliant: true)
