## Fixed Issues

### WR-01: `ls *.patch` under `set -e` silently fails when patches directory is empty

**Files modified:** `.planning/scripts/post-merge-export.sh`
**Commit:** 17cc875
**Applied fix:** Replaced the `ls "${PATCHES_DIR}"/*.patch 2>/dev/null | sort` process substitution with a `find "${PATCHES_DIR}" -maxdepth 1 -name "*.patch" | sort` invocation. `find` exits 0 when no files match, making it safe under `set -e`. The `COMMITS` array population was also removed as part of the WR-02 fix that superseded the index-based loop.

---

### WR-02: `PATCH_FILES` and `COMMITS` arrays may be misaligned, producing wrong manifest rows

**Files modified:** `.planning/scripts/post-merge-export.sh`
**Commit:** 17cc875
**Applied fix:** Removed the `COMMITS` array and the index-based `for i in "${!PATCH_FILES[@]}"` loop entirely. Replaced with a `for patch_file in "${PATCH_FILES[@]}"` loop that extracts the commit hash directly from each patch file's `From ` header line using `grep -m1 "^From "`. Each manifest row is now self-contained and immune to ordering mismatches between `format-patch` output and `git log` output.

---

### WR-03: `PATCHES_DIR` is not created before use — script aborts on first run

**Files modified:** `.planning/scripts/post-merge-export.sh`
**Commit:** 17cc875
**Applied fix:** Added `mkdir -p "${PATCHES_DIR}"` immediately before the `rm -f` line in Step 2, ensuring the directory exists before both the `rm -f` glob expansion and the subsequent `git format-patch --output-directory` call.

---

### WR-04: `rm -f` glob expansion on empty directory passes literal glob string

**Files modified:** `.planning/scripts/post-merge-export.sh`
**Commit:** 17cc875
**Applied fix:** Wrapped the `rm -f "${PATCHES_DIR}"/*.patch` call with `shopt -s nullglob` / `shopt -u nullglob` so that the glob expands to nothing (not a literal string) when no `.patch` files exist. This is Option A from the review suggestion — idiomatic and minimal.

---

### IN-01: Hardcoded branch name `thamw-main` reduces portability

**Files modified:** `.planning/scripts/post-merge-export.sh`, `.planning/scripts/pre-merge-check.sh`
**Commit:** ad47966
**Applied fix:** Introduced a named constant `FORK_BRANCH="thamw-main"` at the top of both scripts, with an inline comment `# IN-01: Personal fork branch name — change here if renamed`. All `thamw-main` references in `git format-patch`, `git log`, and `git diff` invocations were replaced with `"${FORK_BRANCH}"`. The manifest range string was also updated to use the variable.

### IN-02: `git show --name-only --format=""` produces a leading blank line in some git versions

**Files modified:** `.planning/scripts/post-merge-export.sh`
**Commit:** 17cc875
**Applied fix:** Added `| grep -v '^$'` between `git show --name-only --format="" "${hash}"` and `paste -sd ',' -` so that any leading blank line emitted by git is stripped before the comma-join, preventing a spurious leading `, ` in the manifest's "Files Touched" column.
