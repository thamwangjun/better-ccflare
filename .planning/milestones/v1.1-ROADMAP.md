# Milestone v1.1: Extended caching for openrouter models

**Status:** ✅ SHIPPED 2026-05-21
**Phases:** 3–6
**Total Plans:** 11

## Overview

Extended OpenRouter cache injection to the full 4-breakpoint limit with a count guard, added `ttl: "1h"` long-TTL support for agentic sessions, and enabled per-account OpenRouter provider preferences backed by a full data model, REST API, and Dashboard UI. All v1.1 fork patches annotated and audited; pre-merge-check.sh hardened with all new high-risk files.

## Milestone Summary

**Key Decisions:**

- PROV-01: Store `allow_fallbacks` as part of the JSON object (not bare array) to support adding further OpenRouter provider fields without a schema change
- PROV-01: `"provider" in body` field-presence check (not `!body.provider` truthiness) to preserve `body.provider = {}` case
- CACHE-03: `countExistingCacheControlBlocks()` extracted as module-level helper for readability and testability
- CACHE-04: No `ttl:` injected in `transformRequestBody()` — `injectSystemCacheTtl()` in `proxy.ts` handles TTL upgrade (coordinated path)
- PROV-04: Dialog gated on `account.provider === "openrouter"` — non-OpenRouter accounts show no dialog item

**Issues Resolved:**

- Phase 3 `accounts.ts` handler constructs `AccountResponse` from raw SQL (not `toAccountResponse()`) — field and SELECT list patched inline
- Phase 3 `auto-refresh-scheduler.ts` bare `Account` literals missing new field — added `openrouter_provider_preference: null` to all 3 literals
- Phase 4 Plan 01: PG migration gap (CLAUDE.md violation) — ported to both PG paths before test implementation
- Phase 4 Plan 02: 3 of 10 new tests immediately GREEN (negative-contract tests) — documented as expected behavior

**Issues Deferred:**

- Per-request OpenRouter provider selection (`x-better-ccflare-openrouter-provider` header) — deferred to future milestone
- Discard Changes dialog behavior has no formal UAT test (SC-4 gap) — testing gap only, not a functional failure
- Pre-existing 27 Biome lint errors in dashboard React components — untouched per STATE.md constraint

**Technical Debt Incurred:**

- None. All 5 pre-audit tech debt items (TD-1 through TD-5) closed by commit `9be312b9` before milestone close.
