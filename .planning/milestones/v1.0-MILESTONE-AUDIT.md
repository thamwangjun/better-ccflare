### Integration Gaps Found

#### Gap 1: openrouter/provider.ts Missing `// FORK PATCH:` Comment Markers (MAINT-01 / SOP Step 4)

`UPSTREAM_MERGE.md` Step 4 instructs an agent to locate fork changes in `openrouter/provider.ts` by looking for:
```
// fix: normalize Authorization header comparison
// fix: strip /v1 prefix
```
Neither comment exists in `packages/providers/src/providers/openrouter/provider.ts`. The `transformRequestBody` and `extractUsageInfo` overrides are also unmarked.

**Impact:** An agent resolving a conflict in this file cannot use the SOP's "look for FORK PATCH comment" anchor. Prose description of fork changes is accurate; search anchors are missing.
**Severity:** SOP guidance gap — does not affect runtime behavior.
**Affected requirement:** MAINT-01 (SOP accuracy).

#### Gap 2: UPSTREAM_MERGE.md Step 4 Code Example References Non-Existent Fields (MAINT-01 / SOP Step 4)

The SOP example for `openai/provider.ts` conflict resolution shows:
```typescript
cacheCreationInputTokens: usage?.cache_creation_input_tokens ?? promptTokensDetails?.cache_creation_tokens ?? promptTokensDetails?.cache_write_tokens ?? 0,
```

Actual code at `openai/provider.ts` lines 263–266:
```typescript
const cacheCreationInputTokens =
    promptTokensDetails?.cache_creation_input_tokens ||
    promptTokensDetails?.cache_write_tokens ||
    0;
```

Three specific discrepancies:
1. `usage?.cache_creation_input_tokens` — does not exist in local scope (only `json.usage` is in scope)
2. `promptTokensDetails?.cache_creation_tokens` — non-existent field (missing `_input_`)
3. `??` vs `||` — syntactically different (equivalent for token counts, but the example does not match)

**Impact:** An agent following this example literally would emit non-compiling TypeScript during conflict resolution.
**Severity:** Agent-blocking for a real upstream merge conflict on this file.
**Affected requirement:** MAINT-01 (SOP accuracy).
