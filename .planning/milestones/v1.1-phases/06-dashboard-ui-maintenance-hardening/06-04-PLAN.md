== Test 2 (SC-2): Save preference -> proxy injects provider.order ==

Setup: Start an nc echo server to capture the upstream request body. The proxy will forward to this echo server instead of the real OpenRouter upstream.

```bash
nc -lk 9999
```

Configure an OpenRouter account to point at http://localhost:9999 as its custom endpoint (or use whichever non-Anthropic account you use for local proxy testing).

Step 1 - Save a provider preference via the dashboard:
1. Open the dashboard -> Accounts tab.
2. Click the dropdown on the OpenRouter account -> "Provider Preferences".
3. In the dialog, enter `anthropic/claude-3-5-sonnet` in the Provider Order field.
4. Leave "Allow Fallbacks" enabled.
5. Click Save. The dialog should close without error.

Step 2 - Send a request through the proxy for that account:

```bash
curl -X POST http://localhost:10180/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer test" \
  -H "x-better-ccflare-account-id: YOUR_OPENROUTER_ACCOUNT_ID" \
  -d '{"model":"z-ai/glm-4.5-air:free","messages":[{"role":"user","content":"test"}],"max_tokens":10}'
```

Expected in nc output: The JSON body forwarded to the echo server contains:
  "provider": { "order": ["anthropic/claude-3-5-sonnet"], "allow_fallbacks": true }

== Test 3 (SC-3): Clear preference -> proxy stops injecting ==

Step 1 - Clear the preference:
1. Open Provider Preferences dialog for the same OpenRouter account.
2. Delete all text from the Provider Order field (leave it empty).
3. Click Save (the empty field triggers a DELETE call, clearing the preference).

Step 2 - Send another request through the proxy (same curl command as Test 2).

Expected in nc output: The JSON body does NOT contain a "provider" field at all.

IMPORTANT: Do NOT test SC-2/SC-3 by routing through the claude account or by curling the Anthropic endpoint directly. Use a non-Anthropic account with x-better-ccflare-account-id.
