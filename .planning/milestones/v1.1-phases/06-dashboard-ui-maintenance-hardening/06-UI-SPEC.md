| Token | Value | Tailwind Class | Usage in this phase |

| xs | 4px | `gap-1`, `p-1` | Icon margins inside dropdown items (`mr-2` = 8px for icons) |
| sm | 8px | `gap-2`, `p-2` | Inline element gaps, badge padding (`px-2 py-0.5`) |
| md | 16px | `gap-4`, `p-4` | Default card/section padding, dialog content padding |
| lg | 24px | `gap-6`, `p-6` | Not used directly in dialog; gap between major form groups |
| xl | 32px | `gap-8` | Not used in this phase |
| 2xl | 48px | `gap-12` | Not used in this phase |
| 3xl | 64px | `gap-16` | Not used in this phase |

Exceptions:
- Dialog inner content: `space-y-4 py-2` (vertical rhythm: 16px between items, 8px vertical padding within scroll zone) — source: `AccountModelMappingsDialog.tsx` line 123
- Form label-to-input gap: `space-y-1` (4px) — source: `AccountModelMappingsDialog.tsx` line 135
- Dialog footer top margin: `mt-2` (8px) — source: `AccountModelMappingsDialog.tsx` line 174
- Dialog max-width: `sm:max-w-[500px]` — narrower than ModelMappingsDialog (600px) because the new dialog has only two controls. Source: RESEARCH.md code skeleton line 489

Declared weight scale: **2 weights only** — font-normal (400) for body/metadata, font-semibold (600) for labels/headings.

| Role | Light mode value | Dark mode value | Tailwind token | Usage |

| Destructive | `hsl(0 84.2% 60.2%)` | same | `text-destructive` / `bg-destructive` | Delete account action only |

**Accent (#f38020 ccflare Orange) is reserved for:**
- The "set" indicator icon on the "Provider Preferences" dropdown item when `openrouterProviderPreference` is non-null: `text-primary` on the `Settings2` icon
- Focus ring on interactive inputs (`--ring` maps to primary orange)
- Primary Save button background (`Button` with no variant defaults to primary)

Accent must NOT be used for: body text, labels, disabled states, helper text, or neutral badges.

## Interaction Contract

### Dialog Open/Close

- Entry: User clicks "Provider Preferences" in the AccountListItem dropdown menu. Gated: only rendered when `account.provider === "openrouter"`.
- Open: Dialog becomes visible. `useEffect` fires on `account` prop, populating `providerOrder` from `account.openrouterProviderPreference.order.join(", ")` and `allowFallbacks` from `account.openrouterProviderPreference.allowFallbacks`. If preference is null: `providerOrder = ""`, `allowFallbacks = true`.
- Cancel ("Discard Changes"): Calls `onOpenChange(false)`. No state mutation. Dialog closes.
- Save (non-empty order): Parses comma-separated input, calls `onSetProviderPreference(account.id, parsedOrder, allowFallbacks)` → PUT. On success, calls `onOpenChange(false)`.
- Save (empty order): Calls `onClearProviderPreference(account.id)` → DELETE. On success, calls `onOpenChange(false)`.
- Loading state: Both "Discard Changes" and Save buttons are `disabled={isLoading}`. Save button label changes to "Saving..." during in-flight request.
- Error: Catch block calls `console.error`. Dialog remains open (does not auto-close on error). AccountsTab catches and sets `actionError` via the `setActionError(formatError(err)); throw err` pattern.

### Switch Interaction

- Toggle: Radix Switch, `checked={allowFallbacks}`, `onCheckedChange={setAllowFallbacks}`.
- Default state: `true` (ON/checked) — matching API default and CONTEXT.md D-01.
- Label association: `htmlFor="allow-fallbacks"` on Label, `id="allow-fallbacks"` on Switch.
- Switch is always interactive regardless of whether providerOrder is empty (the toggle state is only relevant when a non-empty order is saved; empty order = DELETE regardless of toggle value, per D-02).

### Dropdown Item State Indicator

- When `openrouterProviderPreference` is non-null: icon renders with `text-primary` class (orange), a `"set"` span appears right-aligned with `ml-auto text-xs text-muted-foreground`.
- When null: icon renders without color modifier (inherits default text color). No badge.
- Tooltip (title attribute): Dynamic — shows current order when set, generic description when unset.

## Layout Contract

The new dialog follows the exact layout structure of `AccountModelMappingsDialog`. The only structural difference is the content area:

```
DialogContent (sm:max-w-[500px])
  DialogHeader
    DialogTitle: "Provider Preferences"
    DialogDescription: "Configure OpenRouter provider routing for {name}."
  div.space-y-4.py-2              ← content scroll zone
    div.space-y-1                  ← provider order field group
      Label htmlFor="provider-order"
      Input id="provider-order"
      p.text-xs.text-muted-foreground  ← helper text
    div.flex.items-center.gap-2    ← allow fallbacks row
      Label htmlFor="allow-fallbacks"
      Switch id="allow-fallbacks"
  DialogFooter.mt-2.shrink-0
    Button[variant="outline"]  Discard Changes
    Button                     Save Changes / Saving...
```

The content area does NOT use `overflow-y-auto flex-1` (present in ModelMappingsDialog for its taller grid) because this dialog has only two controls and will not overflow at any viewport size.
