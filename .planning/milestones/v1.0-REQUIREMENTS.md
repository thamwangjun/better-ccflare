## v1 Requirements — All Complete

### OpenRouter Caching

- [x] **CACHE-01**: Non-streaming OpenRouter responses report accurate cache token counts (reads `usage.prompt_tokens_details` instead of Anthropic-native field names that don't exist in OpenRouter responses)

  *Outcome: Validated — `extractUsageInfo` override added to `OpenRouterProvider` reading `prompt_tokens_details.cache_write_tokens` → `cacheCreationInputTokens`. Billing display no longer silently underreported by 5x.*

- [x] **CACHE-02**: `cache_control` ephemeral injection is gated on `anthropic/*` model prefix and applied per-block on the last system message content item (not top-level, which silently fails for Bedrock/Vertex routes)

  *Outcome: Adjusted — implementation deliberately omits the `anthropic/*` model prefix gate per D-04 plan decision (gate would block all non-Anthropic OpenRouter requests). Per-block injection at 3 breakpoints (last tool, last system block, last assistant turn) is correct. REQUIREMENTS.md wording was never updated to reflect this decision — non-blocking cosmetic mismatch.*

  *Outcome: Validated — `// FORK PATCH: cache_write_tokens from prompt_tokens_details (OpenRouter)` comment added on line 262, immediately above `const cacheCreationInputTokens =` declaration.*

- [x] **PATCH-02**: A unit test covers the OpenRouter non-streaming cache token extraction path and fails if the patch is removed or regresses

  *Outcome: Validated — 10-test suite in `openrouter/__tests__/provider.test.ts` includes `expect(usage?.cacheCreationInputTokens).toBe(50)` regression guard. TDD RED commit `c3661ec`, GREEN `db5e5d7`.*
